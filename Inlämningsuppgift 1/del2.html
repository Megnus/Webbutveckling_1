  1. Element: Är en komponent i DOM (dokumentobjektmodellen) för att rendera webbsidor i html. Dessa består i regel av en start- och en sluttagg. Ex: <p>Exempel på paragraph tag</p>

    Attribut: Dessa tillhör html-element och definerar elementets funktion eller hur denna skall renderas. Ex: <p title="example">Title är ett attribut</p>

    Värde: Denna tillhör ett input-element och fungerar annorlunda beroende på vilket element som den finns för. Som regel håller den det värde som finns för elementet och renderas på elementet. Ex: <input type="submit" value="Submit form">

  2. Kommunikationsprotokoll är ett protokoll som säger hur två eller flera parter skall kommunicera. Ett av de många kommunikationsprotokollen som finns är html. Denna har från början haft som syfta att överföra information i form av html-sidor från server till klient.

  3. W3C står för World Wide Web Consortium och består av ett flertal företag samt orginistaioner och har som syfte att skapa en standard för komminikation över webben i form av tekniska protokoll och programvara.

  7. HTTP bygger på modellen förfrågan/svar-förfarande vilket sker mellan klient och server.

  11. WYSIWYG-programvaror och HTML-generatorer kan innebära en hjälp men kunskap om html behövs ändå för att kunna editera denna markup vilket kan vara begränsad vid genererad. Jag tror även att sekundärt genererad html skapar väldigt mycket redundant kod dvs kod som upprepas och i slutändan kan resultera i långsammare rendering. Jag tror även att det kan vara svårt att, vilket det är överlag, att få sekunderärt genererad html att fungera bra för flera webbläsare. Sekundärt genererad html kod kan även vara svår och förvirrande för någon som skall göra förändringar som samtidgt inte har tillgång till den sepecifika programvaran som användas vid tillfället då markup:en skapades.

  12. HTML-dokumentet består av head och body. Head innehåller information om det aktuella dokumentet som tex titeln och information som används av sökmotorer mm. Själva innehållet i dokumentet finner man i body. Den håller alla element som skall renderas i webbläsaren.

  13. HTML-kommentarer kan likt i alla program-, skript och märkspråk vara till nytta för utvecklare. Det kan vid tillfällen behövas då man måste kommunicera med en andra utvecklare som tar del av koden eller att det vid har förflutit så pass lång tid att man glömt varför markup:en kodats på ett visst sätt. Ibland händer det även att man behöver ändra saker som inte är bra men att man inte har tid och behöver återkomma och då kan det med fördel infogas en kommentar som fungerar som en minnesnotering. Ibland vill man även att informationen från koden skall vara kvar men att man vill undvika att denna skall renderas, vilket kan göras genom att kommentera bort denna kod.
